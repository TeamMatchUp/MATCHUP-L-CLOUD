3. User Roles & Access Levels
1. Public Visitor
Can:
Browse events
View fighter profiles
View confirmed matches
See open fight slots
Cannot:
Propose matches
Contact fighters directly
Access dashboards

2. Organiser
Can:
Create events
Define fight slots
Search fighters
Use AI suggestions
Propose matches
View confirmation status
Confirm final match
Cannot:
Edit fighter records
Override coach/fighter declines

3. Coach
Can:
Manage gym profile
Manage fighter roster
Toggle availability
Accept/decline proposals
Submit fighters to events
Cannot:
Confirm fight without fighter approval
Edit event structure

4. Fighter
Can:
Edit own profile
Toggle availability
Accept/decline match proposals
View confirmed fight details
Cannot:
Propose matches
Modify opponent data

4. Primary User Journeys (Max 3 Steps Each)

Journey 1 — Organiser Creates a Confirmed Fight
Create Event → Define Fight Slot
Propose Match (Manual or AI Suggestion)
Coach Accepts → Fighter Accepts → Status = Confirmed

Journey 2 — Coach Accepts Proposal
Receive Proposal Notification
Review Opponent Comparison
Accept → Await Fighter Confirmation

Journey 3 — Fighter Confirms Match
Receive Match Notification
Review Matchup Card
Accept → Match Confirmed

Journey 4 — Public Discovery
Browse Events
View Open Slot or Confirmed Fight
Register Gym or Create Event

Flow Integrity Rules
No direct messaging outside structured proposal.
No status skipping.
No unilateral confirmation.
All transitions logged.
Confirmed fights publicly visible.
Structured execution reinforces trust.

System Psychology
Discovery = lighter UI, open navigation.
Matchmaking = structured UI, tighter data presentation.
Users always know:
Where they are.
What action is primary.
What state the match is in.
No ambiguity.
